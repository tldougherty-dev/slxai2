# Badge contact pages (static)

This folder is generated by `scripts/generate-badge-contacts.mjs`. Each attendee gets:

- `public/contacts/{slug}/index.html` — mobile-friendly contact card with **Add to Contacts** (downloads the vCard).
- `public/vcf/{slug}.vcf` — vCard 3.0 file. **`URL:`** is the **organization website** from the Website column (the main “homepage” in contact apps). **LinkedIn** profile URLs live only in **NOTE** (after the SLxAI line), never as `URL:`. If the Website cell mistakenly contains a LinkedIn link, it is treated like LinkedIn (NOTE only). **NOTE** always starts with `Met at SLxAI Summit 2026 — https://slxai.org`, then any LinkedIn URLs on separate lines.
- `public/badge-qr-links.csv` — Full Name, Title, Company, Slug, Contact URL, VCF URL (for QR or mail merge). Regenerated with the contact pages.

## Source data

1. Export **Badge List SLxAI 2026 - Badge List.csv** with these columns:

   - Full Name  
   - Current Job Title  
   - Company/Organization Name  
   - Country, or City of Residence  
   - Email Address  
   - Phone Number  
   - Website URL  
   - LinkedIn Profile URL  

2. Save or copy it to:

   `data/badge-list-slxai-2026.csv`

   (Replace the sample file in the repo when you have the full list.)

## Local dev URLs (`npm run dev`, default port 8080)

After generating, restart the dev server so `vite.config.ts` can serve `/contacts/{slug}/`:

- Contact card: `http://localhost:8080/contacts/abraham-glasser/`
- Same file (always works): `http://localhost:8080/contacts/abraham-glasser/index.html`
- vCard: `http://localhost:8080/vcf/abraham-glasser.vcf`

## Generate files

From the project root:

```bash
npm run generate-badge-contacts
```

Or with a custom CSV path:

```bash
node scripts/generate-badge-contacts.mjs "C:/path/to/Badge List SLxAI 2026 - Badge List.csv"
```

Optional: set the site origin used in `badge-qr-links.csv` (default `https://slxai.org`):

```bash
set BASE_URL=https://slxai.org
npm run generate-badge-contacts
```

(On PowerShell: `$env:BASE_URL="https://slxai.org"; npm run generate-badge-contacts`)

## Deployment

- **Vite:** Everything under `public/` is served at the site root in dev and copied into `dist/` on `npm run build`. After build, URLs are:

  - `https://YOUR_DOMAIN/contacts/{slug}/`
  - `https://YOUR_DOMAIN/vcf/{slug}.vcf`

- **Any static host:** Upload the contents of `public/contacts/`, `public/vcf/`, and `public/badge-qr-links.csv` to the same paths on your CDN or object storage.

- **Do not commit** real attendee PII if policy requires; add `data/*.csv` to `.gitignore` if needed and generate in CI from a secret artifact.

## Slugs

Lowercase, spaces → hyphens, special characters removed. Duplicate full names get `-2`, `-3`, etc.
